In Short

Elasticounting is a very simple SPA (Single Page Application) example using Elasticsearch as database (based on MEAN stack). You can import and manage your accounting data (Category/Operation) to Elasticsearch then visualize them through Kibana.

Elasticsearch
Many call example with full client Elasticsearch.js API (Ping, search, save/bulk, delete)
AngularJs
Use Route, Controller and Service to use MVC pattern on front-end side
TypeScript
Minimal use of TypeScript for business entity
Boostrap
Get some default style
Node
Out light web server
Other
Back-end is currently empty but open to extension (Node/Express/Elasticsearch)

Follow this project on GitHub: https://github.com/boomcod/elasticounting

Getting Started

1
Make sure you have Node/npm + bower installed
2
In frontend directory, get bower packages: bower install
3
In backend directory, get node packages: npm install
4
Start the server: node server.js
5
Start your elasticsearch instance
6
Go to your browser at http://localhost:3000
7
Copy/Paste some accouting text in Import section
8
Use the Regex to parse the text then click on Send To ElasticSearch

Configuration

You can change elasticsearch setting in /config.js file: host/index naming, ...

Next Step

Install kibana, configure it to bind 'accounting' index and start to visualize!
